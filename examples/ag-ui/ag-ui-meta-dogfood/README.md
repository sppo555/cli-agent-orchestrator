# AG-UI Meta-Dogfood ‚Äî the audit fleet visualizes itself (AC5 task 19.2)

This is the closing-the-loop capture for #458 (F-DF2 / F-OR2): the multi-agent
**supervisor ‚Üí developer ‚Üí reviewer** session ‚Äî the same shape as the CAO fleet
that produced the #458 hardening plan ‚Äî rendered through the **live AG-UI HTTP
stream** and the L2 constructs.

It is the recorded **real-provider evidence** for spec task 19.2: a genuine
orchestration lifecycle (a cross-provider fleet: `kiro_cli` supervisor,
`claude_code` developer, `codex` reviewer) driven through the *production*
event path, observed on `GET /agui/v1/stream`, and folded by
`SupervisorDashboardStream` + `MultiAgentSessionTimeline`.

Keyless and deterministic ‚Äî no external providers, no tmux, no browser ‚Äî so it
runs anywhere CI runs.

## Run

```sh
CAO_AGUI_ENABLED=1 uv run python examples/ag-ui/ag-ui-meta-dogfood/capture.py
# or:  ./examples/ag-ui/ag-ui-meta-dogfood/run.sh
```

## What it proves

- Under **only** `CAO_AGUI_ENABLED`, the real `EventLogPublisher` observer emits
  the fleet lifecycle to the in-process `EventLog`, and those frames are served
  on the real `GET /agui/v1/stream` endpoint (full production path).
- The **privacy boundary holds**: message *bodies* are never emitted ‚Äî the
  delegation entries carry only `sender`/`receiver`/`orchestration_type`
  metadata (see `timeline.json`).
- Three of the four L2 constructs exercise a real workload:
  `MultiAgentSessionTimeline` renders the assign‚Üíhandoff‚Üíhandoff delegation
  chain; `SupervisorDashboardStream` folds the lifecycle rollup.
